微信小程序简介

什么是微信小程序

温馨小程序是一种全新的连接用户与服务的方式，它可以在微信内被便捷的获取和传播，同时具有出色的使用体验。

手机端app的另外一种新的展现形式

无需下载过多占用内存的app，小程序直接打开

小程序必备技术点

  • HTML
  • JavaScript
  • CSS

适用场景

  • 基于腾讯庞大的社交群体，可以为原生app导流
  • 创业公司优先推出小程序 ，开发成本低
  • 当作简单的工具使用，需要在app上频繁crud不适用

微信小程序的简要注册流程

前提准备：

  • 需要有公众号（未认证个人/认证企业）
  • 需要一个邮箱， 1 v 1

完善小程序信息

  • 小程序名称、头像、介绍、类型
  • 绑定开发者，管理员默认，团队成员需要绑定

全局json （app.json)

定义了这个app的要加载的内容（显示的页面）的路径，界面表现，网络超时时间，底部tab 等

pages 属性定义了要加载的页面的路径，第一个默认未首页

window 字段 小程序所有页面的顶部背景颜色，文字颜色定义在这里的。

底部切换栏： 在底部的首页与具体页面之间的切换操作

tabBar 字段 tabBar 配置项指定 tab 栏的表现，以及 tab 切换时显示的对应页面

而text 则代表该栏目的名称

具体操作： 点击底部切换栏的名字，进行跳转：
点击首页跳转到首页，点击 慕课网跳转到 imooc 下的 imooctext.wxml 页面

networkTimeout 可以设置各种网络请求的超时时间

debug 可以在开发者工具中开启 debug 模式，在开发者工具的控制台面板，调试信息以 info 的形式给出

单items 暂时唯一需要注意的就是page对象，里面有一个data对象，以键值对存放，在页面用template渲染

小程序 app 的生命周期

  • onLaunch 第一次初始化会被出发只会被调用一次
  • onShow 小程序在前台会展现
  • onHide 前台到后台的展现（e.g 双击home 点进其他程序时）
  • onError 对错误时做出处理 与 java中gexception 很像
  • 其他： web开发者可以自行定义（全部都是全局）

小程序模块化：

​ （1） 抽离通用方法作为通用函数

​ （2）构建 utils-common 类

定义公共 js 模块：

①创建 utils 目录，并在该目录下创建common.js 作为公共资源，
②在需要调用 common.js 资源的 js （如 events.js）中的page对象上层使用 var common = require(‘ ../utils/common.js’) 对公共资源进行引入
注意： 节点资源的引入只能使用相对路径，不支持绝对路径

引入到Page后，直接使用 common.sayHello(“lee”),也就是 js 页面名.方法名( “参数” )的方式进行调用
