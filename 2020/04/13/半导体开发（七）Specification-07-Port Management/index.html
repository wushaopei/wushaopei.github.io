07. Port Management

07.01. Port Operation Mode Management

07.01.01. CST Operation Mode

根据玻璃类型指示装载机当前的工作方式。 CST操作模式仅适用于特定的设备(仅适用于阵列，CF)。

  • Kind to Kind Mode (CF/Cell Default)： Glass/chip 按产品(种类)类型装入盒内。它不考虑 glasses 的FIFO顺序。卸载 cassette 满时，卸载报告卸载请求事件。
  • Cassette To Cassette Mode(Array Default)： Glass/chip 将储存在相同的槽卸载磁带作为加载磁带。如果卸载机有金属丝盒，玻璃/芯片可以存储到另一个卸载盒槽。工作数据表设备写入。

07.01.01.02. Communication Scenario

07.01.02. Partial Full Mode

Unloader 应该提供部分Full Mode Enable/Disable function。

  • Enable：
    • When Cassette Status = Process Complete，从 port 卸下全部 Cassette 或部分 Cassette 时，设备需要打开部分全开标志。
    • Unloader port 可以 load 部分 cassette 或 空 cassette
  • Disable：
    • When Cassette Status = Process Complete, 所有Cassette 或者 部分 Cassette 从 port 卸载, equipment doesn’t need to turn on Partial Full Flag.
    • 卸载 port 可以 load 空 cassette.

07.02. Port Function Management

07.02.01. Port Type Change

它是根据当前加载在 Port 上的 Casstte 的预期操作对 Port 进行分类: 加载在 Port 上的 Cassette 是否将 Job 输入到 Inline , 或在 Job 以行方式完成后，从 Inline 方式输出作业 , 或在临时行中缓冲作业 .

如果端口上有磁带，则不能更改端口的端口类型。 设备制造商应该提供这一功能。如果端口类型改变，设备应该报告端口的 ‘Load Request’ 事件 .

There are 4 kinds of port types, loader, un-loader, both and buffer. 根据端口的类型，端口模式与端口类型相关。

  • Loading Port： 用于从装入端口的 cassette 中取出 Job 并将其输入到 Inline 中。

  • Unloading Port： 用于将完成其过程的 Job 存储到 cassette 中 , 把 cassette 从 Inline 取出来

  • Both Port： 用于从装入 Port 的 cassette 中取出 Job 并将其输入到 Inline 中，也用于将完成其处理过程的作业存储到 cassette 中，并将 cassette 从 Inline 中取出

  • Buffer Port：To be used to buffer Job in Inline

    • Buffer Type： 指示此 Port 仅用于缓冲目的。可混合多种类型的玻璃。
    • Loader in Buffer Type： 指示此缓冲端口可用于 Glass Inline 输入，如Loading Port。
    • Un-Loader in Buffer Type： 指示此缓冲端口可用于 Glass Inline 输出，如Unloading Port。

07.02.02. Port Mode Change

它是根据端口盒中所包含的眼镜的操作方式对端口进行分类。

显示玻璃类型或判断信息。基于端口类型的含义有一些不同。

如果端口上有盒式磁带，则端口的端口模式不能更改。设备制造商应该提供这一功能。如果端口模式改变，设备应报告端口的“负载请求”事件。(自动换口模式设备除外)

  • Loading Port： 指示装载在 Port 上的 Cassette 中所包含的 Glass 类型

    • TFT Mode： 表示此端口用于处理TFT玻璃。

    • CF Mode：表示此 Port 用于处理CF Glasses。

    • Dummy Mode：指示此 Port 用于将 Dummy Glasses 加工成 Inline

    • MQC Mode：表示此 Port 用于将MQC glassses 加工成 Inline。

    • HT Mode：Indicates that all glasses in this Port would enter HT Chamber in CVD for

      processing

    • LT Mode：Indicates that all glasses in this Port would enter LT Chamber in CVD for

      processing

    • ENG Mode：Indicates that this Port is for processing MQC Glasses into Inline.

    • IGZO Mode：Indicates that this Port is for processing IGZO Glasses into Inline

    • ILC Mode：Indicates that all glasses in this Port would enter the equipment in ILC mode for processing

    • FLC模式:表示该端口的所有眼镜进入FLC模式进行处理

    • Through Dummy Mode：Indicates that this Port is for processing Through Dummy

      Glasses into Inline.

    • Thickness Dummy Mode：Indicates this Port is to fetch Thickness Dummy Glasses into Inline.

    • UV Mask Mode：表示此端口用于将 UV Mask Glasses Inline 加工。

    • Mismatch Mode：用于处理 Array/ CF line 的排序 ，表示此端口用于处理VCR读不匹配的 Glass。

  • Both Port： 这个端口用于 loading 和 un-loading 这两种类型。

    • TFT Mode： 指示此端口用于 loading 和 un-loading TFT玻璃。
    • CF Mode：表示此端口用于 loading 和 un-loading CF 玻璃。
    • Dummy Mode：表示此端口用于 loading 和 un-loading Dummy Glasses。
    • MQC Mode： 表示此端口用于 loading 和 un-loading MQC Dummy Glasses。
    • HT Mode： 指示此端口用于 loading 和 un-loading，将进入CVD中的HT室进行处理。
    • LT Mode： 表示此端口为加载，卸载时将进入CVD中的LT室进行处理。
    • ENG Mode： 说明此端口用于MQC玻璃的装卸加工。
    • IGZO Mode： 表示此端口用于装载和卸载加工IGZO玻璃。
    • ILC Mode：表示此装卸口将进入设备的ILC模式进行处理。
    • FLC Mode：表示此装卸口将进入设备FLC模式进行处理。

    设备仅根据端口类型使用一种端口模式:加载器、卸载器和缓冲区。

    例如，卸载器可能有正常，返工，修理或NG端口和缓冲区可能有加载，卸载，虚拟，刮，返工或缓冲区。

07.02.04. Port Enable Mode Change

端口启用模式意味着将使用该端口。只有当端口上没有磁带时，端口Enabled / Disabled 更改才可用(即 load request status,)。一旦端口状态被设置为禁用，就不应该允许或更改与端口相关的端口模式或磁带状态。

如果端口上有盒式磁带，则端口的端口启用模式不能更改。

设备制造商应该提供这一功能。如果端口启用模式从禁用模式更改为启用模式，设备应报告端口的“负载请求”事件。

  • Enabled Mode： 可以使用此端口。 First Glass Check
  • Disabled Mode： 此端口无法使用。操作员或其他人员禁止使用此端口

07.02.05. Cassette on Port Q Time Change

所有有负载端口的设备应提供端口Q时间功能的磁带。如果卡带等待处理状态超过设置Q时间，设备应自动取消此卡带并发出警告。Q时间应该在触摸屏上设置。

当Q时间值发生变化时，设备应向BC报告。

若Q时间值为0，则表示设备不自动取消卡带，即使卡带已超过Q时间设置。

07.03. Port & Cassette Status Management

07.03.01. Item Define

07.03.01.01. Port Status

Port 状态有4种类型; load request，load complete，Unload Request 和 Unload Complete。

  • Load Request： 端口可以接收盒式磁带，正在等待装入盒式磁带。在触发事件时，应自动增加端口的盒序号。在PLC的R口块中，CST状态下LDRQ后撤，设备将自动将ZR口区数据全部清除，避免残留影响下次下载盒式数据。
  • Load Complete： 一盒磁带已装上了端口。
  • Unload Request：一盒磁带正等着卸下来。
  • Unload Complete： 一盒磁带已经卸载了，端口上没有磁带了。
07.03.01.02. Cassette Status

​ 卡带状态与操作者或设备本身的工艺操作有关。

  • No Cassette Exist： 端口上没有盒式磁带。
  • Waiting for Cassette Data： 磁带当前被装入端口，等待主PLC或操作员的磁带映射数据。设备应报告这一事件后，夹紧盒式磁带和(如果端口有BCR)读取盒式磁带ID和(如果端口有映射单元)感应每个插槽的工作的存在。
  • Waiting for Start Command： Cassette Map 已成功接收，正在等待主PLC或操作员的启动命令
  • Waiting for Processing
    • Loading Port： 开始命令已成功接收，并等待作业取出开始
    • Unloading Port： 已成功接收到Start命令，正在等待将Start插入到磁带中的作业。
  • In Processing
    • Loading Port： 应该先更改为“正在处理中”，然后将T5报告 “Fetch out Job Data” 事件延迟到BC
    • Unloading Port：应先更改为 “In Processing” ，然后将T5报告 “Store Job Data” 事件延迟到BC。
    • Buffer Port：接收到BC的Start命令后，将更改为“In Processing”，然后需要延迟T5报告“Store Job Data”或“Fetch Out Job Data”事件到BC。
    • T5定时器默认值为1秒;范围 1~8秒
  • Process Completed： 过程中的盒式磁带已完成(最后一次玻璃存储在盒式磁带后)或取消，或中止，需要延迟T5定时器才能改变到过程完成状态。
  • Process Paused： 盒式磁带的处理已被BC或操作员(EQ)暂停
  • Re-Map： 操作员在BC OPI改变盒式磁带状态从正在处理重新映射，等待操作员重新选择和下载盒式磁带映射数据。
07.03.01.03. Cassette Sequence No

Cassette Sequence No 是表示盒式磁带的虚构序列号。当该端口的端口状态更改为“Load Request”时，它应该被增加。

07.03.01.04. Operator ID

每个在线的操作员都有唯一的ID。设备必须为操作员ID输入提供屏幕。

07.03.01.05. Cassette ID

如果端口有一个BCR，那么它必须更新项目。BCR夹紧后读取卡带ID。

07.03.01.06. Job Count in Cassette

盒式磁带中的作业计数是盒式磁带在端口装载时的作业计数。当映射单元感知到每个槽中存在作业时，应该更新它。如果工作计数有任何变化，及时报告设备。

07.03.01.07. Job Existence

如果映射单元感知到每个槽中作业的存在，则应针对每个槽对其进行更新并实时刷新。

07.03.01.08. Complete Cassette Dat

它用来表示磁带是如何完成的。

  • Normal Complete： 磁带已正常完成
  • Operator Forced To Cancel： 在工序开始前，操作员已经取消了Cassette
  • Operator Forced To Abort： 在正常的程序完成之前，操作员已经中止了磁带
  • BC Forced to Cancel： 在流程开始前，Cassette 已被BC取消。
  • BC Forced to Abort： 在正常工艺完成之前，卡带已被BC终止。
  • EQ Auto Cancel： Cassette 在开机前已被EQ自动取消。
  • EQ Auto Abort： 在正常工艺完成之前，Cassette 已被EQ自动中止。

07.03.04.08. Event Report Interval Time Scenario

​

(1). 在端口状态 “Load Complete ” 和卡带状态 “ Waiting for Cassette Data ”之间，应延迟1秒，此时(名称:LC等待CST数据时间)可在触摸屏上设置。

(2). 在 “Cassette Map Download” 和卡带状态 “Waiting for Start Command”之间，应延迟1秒。此时(名称:CST下载等待启动时间)可在触摸屏上设置。

(3). 在卡带控制指令 “Cassette Process Start” 和卡带状态 “Waiting for Processing” 之间，应延迟1秒。此时(名称:进程启动等待进程时间)可在触摸屏上设置。

(4). 在卡带状态 “Waiting for Processing” 和卡带状态 “In Processing” 之间，应等待第一次玻璃检查结果事件来改变卡带状态。当最后一块玻璃从盒的装载口离开时，最后一块玻璃标志的工作数据应自动打开。

(5). 在卡带状态变为 “Process Completed” 前，设备应更新卡带各槽位和ZR区域的作业数据

(6). 在卡带状态 “Process Completed” 与bit的BC应答之间，应延迟T6时间。这个时间可以在触摸屏上设置。默认值为0，表示不延迟

(7). 在卡带状态 “Process Completed” 和端口状态 “Unload Request” 之间，应延迟T5时间。T5时间可以在触摸屏上设置。

(8). 在端口状态 “Unload Complete” 和卡带状态 “No Cassette Exist” 之间，应延迟1秒。此时(名称:UC到No CST存在时间)可在触摸屏上设置。

(9). 在卡带状态 “No Cassette Exist” 和端口状态 “Load Request” 之间，应延迟1秒。此时(名称:从无CST到LR时间)可在触摸屏上设置。

07.03.04.09. Cassette Process Completed Interval Time Scenario

​

(1). 在01号卡带和02号卡带之间完成卡带加工的间隔时间至少为6秒。设备应提供触摸屏或PC上的设置。默认值是6秒。

(2). 当端口01和端口02的卡带同时完成时，设备应报告一个端口并保持至少6秒，然后再报告另一个端口，无论端口类型。

07.04. Cassette Control Command

07.04.01. Command Define

​ 盒式磁带控制指令用于控制设备如何处理端口上的盒式磁带。

  • Cassette Map Download : 当设备请求一个端口的盒式磁带映射数据时，BC应将数据下载到设备。BC将盒式映射数据写入已请求该数据的设备的本地PLC中。此时，BC向设备发送此命令。设备可根据卡带图数据对卡带进行处理

  • Cassette Process Start： 如果从BC接收的盒式磁带映射数据正常，设备应向BC请求盒式磁带进程启动命令。当从BC收到此命令时，设备可以启动卡带。

  • Cassette Process Start By Count： 有些设备需要特定的工作数量才能启动。此命令用于这些类型，并包含要启动的作业数量。操作与卡带进程启动命令相同。

  • Cassette Process Pause： 卡带可以暂停。如果接收到此命令，设备必须暂停进程。如果机器人(或传送带)当前正在提取或存储作业到磁带，则在设备将磁带状态切换为暂停之前，应完成提取或存储操作。

  • Cassette Process Resume： 如果磁带进程暂停，设备可以通过此命令重新启动进程。

  • Cassette Process Cancel： 如果磁带没有被处理，BC可以取消该过程。BC使用这个命令。设备将卡带从端口上松开。

  • Cassette Process Abort： 如果正在处理磁带，BC可以使用此命令终止进程。如果机器人(或传送带)当前正在提取或存储作业到磁带，则在设备将磁带状态切换到abort之前，应完成提取或存储操作。

  • Cassette Reload/Load： 当端口状态= “ 3:Unload Request “，卡带状态= “ 7:Process Completed “时，BC/OP可以将Reload命令设置为设备，如果设备接受命令，则端口状态变为” 4:Unload Complete “，卡带状态变为7:No Cassette Exist “。

    端口状态将更改为“4:卸载完成”，盒式磁带状态将更改为“1:不存在盒式磁带”

    端口状态变为“1:Load Request”，卡带状态变为“1:No Cassette Exist”，等待BC/OP将Load命令设置到设备。

  • Cassette Re-Map：用于重新选择使用的函数。设备应将卡带状态更改为“重新映射”，并等待BC重新下载新的卡带映射数据到设备。

07.04.02. Cassette Map Data

它是盒式磁带中每个插槽的工作数据。BC可以将要求盒式映射数据的设备的数据写入本地PLC。

07.04.03. Cassette Control Command Return Code

它是BC的盒式磁带控制命令的返回代码。设备必须根据命令返回代码进行应答。

※ Note：

Return Code Description：

1：Command OK, 意思是设备接受指令

2: Command Error（命令错误），意思是BC命令错误，需要警告，设备不能自动取消/中止卡带。它应该再次等待BC下载命令，直到操作员或BC手动取消。

3：CSTID is Invalid （CSTID是无效的(包括密盒ID)），这意味着CSTID的BCR读取是非法的，需要发出警告，设备不能自动取消/终止盒式磁带。它应该等待操作符或BC取消。

4：Recipe Mismatch（配方不匹配），意味着配方ID的作业数据不能在设备中使用，需要报警，设备不能自动取消/中止卡带。它应该等待操作符或BC取消。设备应按配方ID检查模式，配方自动切换模式检查配方ID。

5：Slot Information Mismatch （槽信息不匹配），即设备作业存在内容与BC下载的作业存在内容不一致，需要发出警告，设备不能自动取消卡带。它应该等待操作符或BC取消。

(如果没有地图传感器就不要检查)

6：Job Type Mismatch（作业类型不匹配），即作业类型的作业数据与端口模式不同。设备需要提供作业类型不匹配的自动取消启用/禁用按钮。如果启用，卡带应自动取消，如果禁用，设备应发出警告并等待操作员或BC取消

7：Cassette Setting Code Mismatch (BEOL), 仅对于Cell，其设备卡带设置代码与BC下载的卡带设置代码不同，需要发出警告，设备不能自动取消卡带。它应该等待操作符或BC取消。

8：Already Received（已经收到），意思是设备多次收到来自BC的相同指令，需要发出警告，设备不能自动取消卡带。它应该等待操作符或BC取消。

9：Other Error（其他错误）, 它的意思是BC下载命令，设备检查后，它的其他错误发生，它需要提出警告，设备不能自动取消卡带。它应该等待操作符或BC取消。

10：Product Type Mismatch（产品类型不匹配 ）, 它的意思是设备最后的产品类型不同于BC下载的产品类型。设备需要提供产品类型不匹配自动取消启用/禁用按钮。如果启用，卡带应自动取消，如果禁用，设备应发出警告并等待操作员或BC取消。

11：Group Index Mismatch （组索引不匹配），仅用于单元格，如果此功能启用，设备不能混合不同的组索引 glass 。

12：Product ID Mismatch （产品ID不匹配 ），仅对于Cell，如果此功能启用，设备不能混合不同的产品id glasses.

07.05. First Glass Check

Loading Port 或 Both Port 收到 Cassette Control Command 后，Cassette Status 变为“Wait for Process”，在第一次从Cassette 带取出 Glass 前，EQP 需向BC报告“First Glass Check ” event 。BC会将检查结果回复给设备。如果结果为“OK”，则可立即从 Cassette 中取出第一块 Glass ;如果结果为“NG”，设备需要自动取消 Cassette 。如果BC没有回复结果，说明T1超时，设备需要重新自动重试。重试计数默认值为3，每次重试间隔时间默认值为30秒。retry count 和 retry count 可以在 Touch Panel 上修改。如果已达到重试计数但BC仍不回复 EQP 结果，设备需要自动取消 Cassette .

07.06. Changer Plan Request Report

仅适用于Changer Mode 使用。

当 source cassette 加载到 loading port 时，设备应自动向BC请求 “ Changer Plan ”。BC将对设备的更换计划进行回复。BC答复后，设备应按照更换计划要求CST到 unloading port ，然后 transfer Job 到指定的 target cassette 。

当更换计划完成，但 source cassette 仍有 job 时，设备应再次向BC请求“ Changer Plan ”，直到 source cassette 为空(所有 job 已取出)。

  • Casette Map Flag：
    • 0 No Require (Cassette Map Empty)： Changer Plan Status = No Plan, set this flag = 0.
    • 1 Require (Cassette Map Exist)：Changer Plan Status = Start, set this flag = 1

07.07. Changer Plan Status Report

仅适用于换模使用。

在切换模式下，设备应向BC报告 “Changer Plan Status ” 如下:

  • Plan Status：
    • 0 No Plan： 设备不执行 “Changer Plan”;
    • 1 Start： 设备启动执行 “Changer Plan”;
    • 2 End： 设备已经完成 “Changer Plan”;
    • 3 Ready：设备获得 “Changer Plan” 并准备执行 “Changer Plan”.
