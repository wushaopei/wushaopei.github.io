04. Data Communication

PLC存储器的数据区域分为两个区域，即 Local Area 和 Master Area 。Local Area 用于从 Local PLC 向 Master PLC 发送数据，而 Master Area 用于从 Master PLC 向 Local PLC 传输数据。每个区域的项目根据报告或命令的特征分为几个类型，每个项目类型如下所示。

  • Local Area
    • Status : 表示设备、端口、PLC当前状态的项。
    • Event : 表示设备生成的事件的项
    • Request : 设备请求对BC做任何动作的许可。
    • Command Reply : 设备响应主PLC的命令 (BC)
  • Master Area
    • Status : 表示BC和主PLC状态的项。
    • Command : 可以发送到本地PLC的项
    • Event Reply : 从设备中针对事件进行回复的项。
    • Request Reply：根据设备的要求进行回复的项目

04.01. Link Relay, Link Register Relation

  • 当事件发生或某些数据发生变化时，本地PLC必须将数据写入特定的链路寄存器(W区)，然后打开相关的本地链路继电器(B区)触发事件
  • 当链路继电器的数据发生变化时，主PLC应读取相关的链路寄存器区域，然后打开自己的链路继电器进行应答，应答对应于本地继电器。
  • 当主PLC的响应是OK时，本地PLC关闭其链路继电器。
  • 当本地PLC的链路继电器断开时，主PLC断开与本地PLC的链路继电器相对应的链路继电器

04.02. EQP Event Report Timing Chart

04.02.01. Event Report Timing Chart (Local to Master)

如图，是机台报到BC 的流程。

第一步，Local 会先写Word Area ，然后Bit on 起来。

第二步，BC 监测到Local 的BIT on 起来后，会在对应的Word 去读取数据（Data Reading)，读取完数据后会进行处理。处理完成好，会进行 Reply on 起来回复。

第三步，机台收到回复后，机台的 Event Report 会 off 。BC 收到机台的 off 后，BC的 Event Report Reply 也会 off.

以上就是一个事件的完整流程。

关于CIM MODE ：

CIM MODE 如果是开着的，那么就按照正常流程去走即可；

如果CIM MODE是关掉的，就不会有Local On BIT 和 Master 检测 Bit 变化的那一步，机台的Report 会一直off ，但是数据还是要写入到 Word Area 的。

为什么要这么做？

在机台、机况等比较重要的状态，包括 port 信息等，如果该流程是在CIM MODE off 的状态下执行的，它实时的更新到Word Area ，在它开启之后，或者BC 重启之后，会重新捞一遍数据。我们会给他更新到最新的状况，而不是说现场发生了什么我们还不知道。

04.02.02. Request Timing Chart (Local to Master)

① 将数据输入链路寄存器(W区)后，本地PLC打开链路继电器(B区)。

② Local PLC加载计时器检查超时。(T1:4秒)-本地PLC在此期间不报告任何使用此链路继电器的其他事件。

③主PLC读取数据，然后打开链路继电器(B区)。

④ 本地PLC读取主PLC的数据并关闭链路继电器(事件位)，然后继续下一步动作

⑤主PLC关闭链路继电器(事件应答)，定时器复位后。主PLC在T2时间内扫描本地PLC的链路。如果超时，主PLC处理错误例程(T2超时时间为2秒)

  • EQ厂商提供T1定时器数值设定功能。(默认值= 4秒)
  • 本地PLC在链路寄存器中保存一次设置的报表数据，直到生成新的报表数据。

流程：

① 机台将数据写入到W区，也就是Data中，然后Bit Area on 起来

② BC 收到机台 on 起来的消息，会先去读 机台的 W区；并进行逻辑运算、数据处理，将处理结果写入BC的 W区（Return Code）,然后Bit 也 on 起来

③ 机台收到BC on 起来的消息，就去读BC的 W Area ，读取完成后就将Bit off

④ BC 收到 机台 Bit Are off ,也进行 off。

04.02.03. Consecutive Event Report Timing Chart (Local to Master)

​ 当本地PLC有多个连续数据要使用一个事件位(如Alarm或Job)报告给主PLC时，上述时序图指定了数据报告事件的过程。

① 如果本地PLC有数据(命名数据A)要报告给主PLC，则在缓冲区中设置数据A

② 如果本地PLC目前没有其他先前的数据报告给主PLC，则移动数据到缓冲区中设置一个连接寄存器区域并报告给主PLC。

③ 如果本地PLC有新的数据(命名数据B)报告给主PLC，本地PLC在缓冲区中设置数据B。如果此时有之前的数据(即数据A)正在报告给master，本地PLC等待，直到数据报告过程结束

④ 主PLC关闭链路继电器(事件应答)后，本地PLC等待0.5秒(T3:0.5秒)，然后本地PLC将数据B移动到链路字区，并报告给主PLC

  • EQ厂商提供T3定时器值设定功能。(默认值= 0.5秒)

04.03. BC Command Reply Timing Chart

04.03.01. BC Command Timing Chart (Master to Local)

BC Notice Area is related to Equipment Command Reply Area （BC通知区与设备命令应答区相关）

①将数据设置到主PLC的链路寄存器区域后，主PLC再打开链路继电器(命令)

②主PLC加载定时器检查超时。(T1: 4秒)

③本地PLC读取主区域的数据。读取后，本地PLC打开本地PLC的链路继电器(命令应答)

④主PLC在命令应答时关闭链路继电器(命令)。如果在T1超时内本地PLC没有打开应答位，则主PLC处理错误例程

⑤本地PLC关闭链路继电器(命令应答)，定时器复位后。本地PLC在T2时间内扫描主PLC的链路继电器。如果超时(T2)发生，本地PLC处理错误例程。

  • EQ厂商提供T2定时器数值设定功能。(默认值= 2秒)

04.04. Time Out Error

04.04.01. T1 Time Out (Master ↔ Local)

① 将数据输入链路寄存器后，本地或主PLC打开链路继电器(事件或命令位)

② 本地或主PLC加载定时器检查超时。(T1:4秒)如果伙伴的响应不存在，则本地或主PLC超时。

③ 在定时器复位后，关闭连接继电器(事件或命令位)。

04.04.02. T2 Time Out (Master ↔ Local)

① 将数据读入链路字后（W区），打开链路中继(命令或事件应答Bit区)

② 加载计时器来检查超时 。(T2:2分钟)。如果伙伴的响应不存在，则本地或主PLC进行超时。

③ 在定时器复位后，关闭连接继电器(事件或命令应答位)

04.04.03. T3 Delay Time (Master ↔ Local)

① 主PLC关闭链路继电器(事件应答)后，本地PLC等待0.5秒(T3:0.5秒)，然后本地PLC将数据B移动到链路字区，并报告给主PLC

  • EQ厂商提供T3定时器值设定功能。(默认值= 0.5秒)

04.04.04. Tn Time

  • Event Report Timing Chart (Local to Master) us Bar

① 将数据设置到链接寄存器(W区)后。本地PLC开启链路继电器(事件位)

② 当数据写入链接寄存器时。链路继电器(事件报告位)必须延迟0.2秒才能打开。Tn定时器的最大值是1秒。

04.05. Data Expression

04.05.01. ASCII Type

  • 使用8位ASCII码。
  • 用ASCII类型的空白(H20)值填充空值。
  • 字内字符间顺序:低字节→高字节顺序
  • 词序:低词序→高词序
  • 字符和数字数据格式:标准字符格式

Example：

  • Character data：(Ex.) ‘R’ → H2052
  • Character data：(Ex.) ‘I1’ → H3149 (H4931 is an error)
  • Character data：(Ex.) ‘123’ → H3231 2033
  • Character data：(Ex.) ‘2AA00101 ‘ → H4132 3041 3130 3130 2020 2020
