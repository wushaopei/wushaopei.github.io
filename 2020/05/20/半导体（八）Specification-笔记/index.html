发表于 2020-05-20 | 分类于 半导体 | | 阅读次数: | 阅读数

当设备状态由Idle 变为 Setup的时候 设备会触发哪些事件上报给BC

如果设备状态改变为 setup，设备应提前报警. 上报： AlarmStatusChangeReport:报警状态上报。

设备超过 no processing time 时间，会发生什么？

设备是空转时，如果设备超过 no processing time 时间，则设备应报告 Idle。(无处理时间=0:不检查无处理时间)(默认值:Tact Time * 2)

Tack Time: 设备上片到出片的时间为一次 Tack Time.

请简述Send Wait Time Out Report功能逻辑

​ 当上游设备将玻璃转移到下游设备，但下游设备没有开始接收上游设备的玻璃，当超过发送等待计时时间，则上游设备需要向BC报告此事件。

一个设备有哪些 Common Event，并对各个Event做简单功能说明

如上图，不限于上图中的数量；

设备间进出

  • Receive Job Data Event 当设备从上游设备接收作业时，应报告此事件
  • Send Out Job Data Event 当设备向下游设备发送作业时，应报告此事件。

设备内部 unit 进出

  • Store Job Data Event 当设备将 Job 存储到一个unit 或 cassette 时，应使用 port 和 unit ID 报告此事件。
  • Fetch Out Job Data Event 当 EQP 从一个 unit 或 cassette 中取出 Job 时，应使用 Port 和 unit ID 报告该 Event 。

其他操作

  • Remove Job Event Mode 当 Job Data 刚刚从 EQP 中删除时，将报告已删除 Job的 Job Data .
  • Job Count Report 实时操作，作业计数是报告整个设备的作业计数(不包括卡带)。

常用的 参数：

  • Unit or Port：用于区分从单元或端口取出的产品。
  • Unit No： 单元的单元号。如果设备有槽位，设备应更新槽位号
  • Port No： 工作取的磁带的端口号。设备应更新槽位号。
  • Slot No： Slot No. 如果单位没有槽位，这个项目应该被清除。

请简述CIM Message功能与设备相关的操作联动

声明：CIM报文用于通知特定设备的操作员特定信息。当接收到CIM报文时，设备必须将其显示给操作员，并在本地保存历史数据。

  • CIM Message Set Command：往设备发一条消息，然后显示。
  • CIM Message Clear Command：发消息后，让设备把发送的消息清除
  • CIM Message Confirm Report：发消息后向设备手动确认
  • Display Least 5 CIM Message：屏幕显示5个Message
  • Display Least 20 History CIM Message：至少保留20条历史记录的Message

Terms Definition

1. Job

1.1 Substrate Type

在Job中，玻璃的类型共有四种：Glass 、cut、chip、Cassette

1.2 Glass ID :

每个Glass有一个Glass ID, Glass ID是唯一的，每一片玻璃只能有一个Glass ID, 全场唯一，以后也不会有重复的。

Glass ID 如果切成小板，则会对每个小板标记为01、02这样的标记作为区分。

1.3 Cut ID

玻璃经过第一次切割操作后的 ID。

1.4 Chip ID

经过第二次切割操作后，glass 变成更小的一块，称为chip (它也被称为细胞或面板)。它的ID被称为Chip ID。

1.5 Cassette Sequence No

​ Cassette Sequence No 代表的是 Cassette 的ID 。每上一个新的 Cassette，就会把 Cassette Sequence 进行加一。

1.6 Job Sequence No

​ Job Sequence 的意思是Cassette 的层数，或者说当前是第几层。

1.7 Chip Position(芯片定位)

​ 在第一次或第二次切割后，每个芯片在玻璃中的起始位置

1.8 Job Type

​ Job Type 是用来区分 cassette 或 glass 的种类。 它由Normal 、Dummy 等组成。

1.9 Job Judge

​ 它表示对 Job 信息的判断 . 可能的结果包括： OK, NG, Reject, Rework等。

1.10 OXR Information

​ OXR信息用于识别每个芯片的判断结果，记录在检测设备中。OXR信息有R, L, O (OK)， X (NG)和F (Down Grade)状态。

1.11 Job Grade

​ 玻璃的等级，记录在检验设备上。等级有 OK, NG, RP, PC, MD, CD, ID, DR, RW 等。

1.12 Tracking Data

​ 跟踪用于记录玻璃工艺流程的数据

2、 Equipment

2.1 Inline Mode

​ Inline Mode 分上下游两种，一种是upstream Inline ，另一种是 downstream Inline 。

2.2 CIM Mode

​ CIM Mode 是指定设备与BC之间的通信关系。有 CIM ON 、CIM OFF 两种

2.3 Equipment Status

​ Equipment Status 显示设备当前的运行状态。它由 Setup, Stop, Pause, Idle and Run构成 .

2.4 Unit Status

​ Unit Status 显示单元的当前操作状态。它由 Setup, Stop, Pause, Idle and Run构成 .

2.5 Alarm Status

​ 每个Alarm可以在两种状态之一: set 和 clear 。

2.6 Alarm ID

​ Alarm 标识符

2.7 Alarm Level

​ Alarm 的级别。它由 Warming 和 Error 组成 。

Warming 只是起到提示作用，而 Error 的出现可能导致宕机。

2.8 Processing Time

​ 它是 EQP 或 Unit 中一项工作的实际处理时间。它不包括在上游设备和下游设备之间转移所花费的时间。

2.9 VCR No

​ VCR No用于区分设备中唯一的VCR。

2.10 lonizer-Fan No.

​ lonizer-Fan No. 用于区分设备中唯一的Ionizer-Fan。

3、Port

3.1 Port Type

​ Port 的分类，有4种类型:Loading Port, Unloading Port, Both Port and Buffer Port 。

3.2 Port Mode

​ 它是根据已包含或将在 Port 上的 Cassette 中排序的 Job 判断对 Port 进行分类。

​ 根据 Port 中所包含的 Glass 的操作方式对 Port 进行分类

3.3 Port Transfer Mode

​ 它表示与端口连接的AMHS机器的类型，用于将 Cassette 从 Port 传送到/或从 Port传送 .

3.4 Port Enable Mode

​ 表示 Port 是否可以使用.

3.5 Port Status

​ 表示一个 Port 是否可以用来装载 Cassette .

​ 共有四种状态： Load Request, Load Complete, Unload Request and Unload

Complete

3.6 Port Grade

​ Port 等级设置结果报告。检查后，出货标准为玻璃放Cassette

3.7 Cassette Status

​ 它指示 Port 上 Cassette 的处理状态。

​ 有以下几种状态： No Cassette Exist、Waiting for Cassette Data、Waiting for Start Command、 Waiting for Processing 、 In Processing、Process Completed、Process Paused、 Re-Map、

3.8 Cassette ID

​ 特定的ID被分配给单独的 Cassette，称为 Cassette ID。它可以被读取条形码阅读器(BCR)，可由触摸屏上的特殊操作员进行修改。

Data Communication

1、EQP Event Report Timing Chart

1.1、EQP Event Report Timing Chart （设备上报）

如图，是机台报到BC 的流程。

第一步，Local 会先写Word Area ，然后Bit on 起来。

第二步，BC 监测到Local 的BIT on 起来后，会在对应的Word 去读取数据（Data Reading)，读取完数据后会进行处理。处理完成好，会进行 Reply on 起来回复。

第三步，机台收到回复后，机台的 Event Report 会 off 。BC 收到机台的 off 后，BC的 Event Report Reply 也会 off.

1.2、Request Timing Chart (Local to Master)（设备请求）

流程：

① 机台将数据写入到W区，也就是Data中，然后Bit Area on 起来

② BC 收到机台 on 起来的消息，会先去读 机台的 W区；并进行逻辑运算、数据处理，将处理结果写入BC的 W区（Return Code）,然后Bit 也 on 起来

③ 机台收到BC on 起来的消息，就去读BC的 W Area ，读取完成后就将Bit off

④ BC 收到 机台 Bit Are off ,也进行 off。

1.3 Consecutive Event Report Timing Chart (Local to Master) （连续上报事件）

​ 当本地PLC有多个连续数据要使用一个事件位(如Alarm或Job)报告给主PLC时，上述时序图指定了数据报告事件的过程。

① 如果本地PLC有数据(命名数据A)要报告给 BC ，则在 Buffer 中设置数据A

② 如果本地PLC目前没有其他先前的数据报告给 BC，则移动数据到 Buffer 中设置一个Word Area 并报告给主PLC。

③ 如果本地PLC有新的数据(命名数据B)报告给 BC，本地PLC在 Buffer 中设置数据B。如果此时有之前的数据(即数据A)正在报告给master，本地PLC等待，直到数据报告过程结束

④ 主PLC关闭Bit Area(事件应答)后，本地PLC等待0.5秒(T3:0.5秒)，然后本地PLC将数据B移动到Word Area ，并报告给主PLC

  • EQ厂商提供T3定时器值设定功能。(默认值= 0.5秒)

2、BC Command Reply Timing Chart

BC Command Timing Chart (Master to Local)

①将数据设置到主PLC的Word Area 区域后，主PLC再打开Bit Area(命令)

②主PLC加载定时器检查超时。(T1: 4秒)

③本地PLC读取主区域的数据。读取后，本地PLC打开本地PLC的Bit Area(命令应答)

④主PLC在命令应答时关闭Bit Area(命令)。如果在T1超时内本地PLC没有打开应答位，则主PLC处理错误例程

⑤本地PLC关闭Bit Area(命令应答)，定时器复位后。本地PLC在T2时间内扫描主PLC的Bit Area。如果超时(T2)发生，本地PLC处理错误例程。

  • EQ厂商提供T2定时器数值设定功能。(默认值= 2秒)

3、Time Out Error

T1 Time Out (Master ↔ Local)

① 将数据输入Word Area 后，本地或主PLC打开Bit Area(事件或命令位)

② 本地或主PLC加载定时器检查超时。(T1:4秒)如果伙伴的响应不存在，则本地或主PLC超时。

③ 在定时器复位后，关闭连接继电器(事件或命令位)。

T2 Time Out (Master ↔ Local)

① 将数据读入链路字后（W区），打开链路中继(命令或事件应答Bit区)

② 加载计时器来检查超时 。(T2:2秒)。如果伙伴的响应不存在，则本地或主PLC进行超时。

③ 在定时器复位后，关闭连接继电器(事件或命令应答位)

T3 Delay Time (Master ↔ Local)

① 主PLC关闭Bit Area(事件应答)后，本地PLC等待0.5秒(T3:0.5秒)，然后本地PLC将数据B移动到链路字区，并报告给主PLC

  • EQ厂商提供T3定时器值设定功能。(默认值= 0.5秒)
Tn Time

  • Event Report Timing Chart (Local to Master) us Bar

① 将数据设置到 W 区(W区)后。本地PLC开启Bit Area(事件位)

② 当数据写入 W 区时。Bit Area(事件报告位)必须延迟0.2秒才能打开。Tn定时器的最大值是1秒。

Port Management

1、Port Operation Mode Management

1.1 CST Operation Mode

根据 Glass 类型指示装载机当前的工作方式。

  • Kind to Kind Mode (CF/Cell Default)
    • Glass/chip 按产品(种类)类型装入盒内。
  • Cassette To Cassette Mode(Array Default)
    • Glass/chip 将储存在相同的 Unload Cassette 作为加载磁带。入卡数与出卡数相同

1.2 Partial Full Mode（部分Full模式）

Unloader 应该提供部分Full Mode Enable/Disable function。

  • Enable
    • 当 Cassette Status = Process Complete，从 port 卸下全部 Cassette 或部分 Cassette 时，设备需要打开 Partial Full Flag
  • Disable
    • 当 Cassette Status = Process Complete, 所有Cassette 或者 部分 Cassette 从 port 卸载,.设备不需要打开 Partial Full Flag。

2、Port Function Management

2.1 Port Type Change

根据当前加载在 Port 上的 Casstte 的预期操作对 Port 进行分类：

  • Loading Port： 用于从装入端口的 cassette 中取出 Job 并将其输入到 Inline 中。
  • Unloading Port： 用于将完成其过程的 Job 存储到 cassette 中 , 把 cassette 从 Inline 取出来
  • Both Port： 用于从装入 Port 的 cassette 中取出 Job 并将其输入到 Inline 中，也用于将完成其处理过程的作业存储到 cassette 中，并将 cassette 从 Inline 中取出
  • Buffer Port：用于 in Inline buffer 的 Job

2.2 Port Mode Change

根据 Port 中所包含的 Glass 的操作方式对 Port 进行分类；主要分为：

  • Loading Port : 指示装载在 Port 上的 Cassette 中所包含的 Glass 类型
  • Both Port： 这个端口用于 loading 和 un-loading 这两种类型。

2.3 Port Transfer Mode Change

它指出了在端口和AMHS之间传输磁带的方式。有3种模式 :

  • MGV Mode： 与 MGV 的 Port 接口
  • AGV Mode： 与 AGV 的 Port 接口
  • Stocker Inline Mode：与 Stocker 的 Port 接口

2.4 Port Enable Mode Change

Port Enable Mode 意味着将使用该 Port 。

有两种Mode:

  • Enabled Mode： 可以使用此端口。 First Glass Check
  • Disabled Mode： 此端口无法使用。操作员或其他人员禁止使用此端口

2.5 Cassette on Port Q Time Change

所有有 Load Port 的 EQP 应提供 Port Q时间功能的 Cassette。

  • 当Q时间值发生变化时，设备应向BC报告。
  • 若Q时间值为0，则表示设备不自动取消卡带，即使卡带已超过Q时间设置。

2.6 Cassette on Port Q Time Change Command

当设备接收到此事件时，应将Q时间值更改，并将回复结果改为BC。

  • OK：重新计数卡带的端口Q时间。
  • NG：保持原始磁带在Q端口的时间。

2.7 Port Down

当端口有错误不能使用时，设备应触发此功能到BC。

  • Normal：Port is OK
  • Down： 发生错误时端口不能使用。

2.8 Port Type Change Command

当设备收到此命令时，应将结果回复到BC并更改端口类型。如果端口上有 Cassette，则不能更改 Port 的 Port Type。

  • OK：更改端口类型
  • NG：不能更改端口的端口类型

2.9 Port Mode Change Command

当设备收到此命令时，应将结果回复到BC并改变端口模式。如果端口上有磁带，则端口的端口模式无法更改

  • OK ：更改端口的端口模式。
  • NG ：不能改变端口的端口模式

2.10 Port Transfer Mode Change Command

当设备收到此命令时，应将 reply 结果返回给BC，并改变端口传输模式。如果端口上有磁带，则端口的端口传输模式不能改变。

  • OK： 改变端口的传输方式。
  • NG : 无法更改端口的传输模式。

2.11 Port Enable Mode Change Command

BC还可以设置端口类型的改变。当设备收到此命令时，应将应答结果返回给BC并更改端口启用模式。如果端口上有磁带，则该端口无法更改。

  • OK： 改变端口的启用端口
  • NG： 不能更改端口的启用端口

3、Port& Cassette Status Management

3.1 Item Define

3.1.1 Port Status

Port 状态有4种类型; load request，load complete，Unload Request 和 Unload

3.1.2 Cassette Status

同上面的 Cassette Status

3.1.3 Cassette Sequence No

Cassette Sequence No 是表示盒式磁带的虚构序列号。

3.1.4 Operator ID

每个在线的操作员都有唯一的ID。

3.1.5 Cassette ID

同上

3.1.6 Job Count in Cassette

Cassette 中的 Job Count 是 Cassette 在 Port Load 时的 Job Count。

3.1.7 Job Existence

如果映射单元感知到每个槽中作业的存在，则应针对每个槽对其进行更新并实时刷新。

3.1.8 Complete Cassette Dat

它用来表示磁带是如何完成的。

  • Normal Complete： 磁带已正常完成
  • Operator Forced To Cancel： 在工序开始前，操作员已经取消了Cassette
  • Operator Forced To Abort： 在正常的程序完成之前，操作员已经中止了磁带
  • BC Forced to Cancel： 在流程开始前，Cassette 已被BC取消。
  • BC Forced to Abort： 在正常工艺完成之前，卡带已被BC终止。
  • EQ Auto Cancel： Cassette 在开机前已被EQ自动取消。
  • EQ Auto Abort： 在正常工艺完成之前，Cassette 已被EQ自动中止。

3.2 Port & Cassette Status Change

3.2.1 Normal Scenario

上卡流程：

在没有 Cassette 的时候， Port 会启动并发出请求。

1、没Cassette时（No Cassette Exist），port发送请求命令要一个卡夹（Load Request）；

2、拿到卡夹后（Load Complete），等待BC校验卡夹数据（Wait For Cassette Data）；

3、BC校验数据成功时，BC发送消息给设备（Cassette Map Data Download Complete）；

4、设备等待开始（Wait For Start Command）；

5、BC发送启动命令给设备（Start Command），设备等待制程（Waiting For Processing）；

6、BC给设备下开始命令（Process Start）设备正在制程（In Processing）；

7、正常制成 （Normal Complete），设备制程完成（Process Complete）；

8、等待T5 Time时间（1-8秒），port请求发送退卡指令（Unload Request）；

9、退卡完成（Unload Complete）

4、Cassette Control Command

4.1 Command Define

Cassette Control Command 用于控制设备如何处理 Port 上的 Cassette。

  • Cassette Map Download
  • Cassette Process Start
  • Cassette Process Start By Count
  • Cassette Process Pause
  • Cassette Process Resume
  • Cassette Process Cancel
  • Cassette Process Abort
  • Cassette Reload/Load
  • Cassette Re-Map

4.2 Cassette Map Data

它是 Cassette 中每个插槽的 Job Data

4.3 Cassette Control Command Return Code

它是BC的 Cassette Control Command 的 Return Code。设备必须根据命令 Return Code 进行 Reply .

Return Code Description：

  • Command OK
  • Command Error
  • CSTID is Invalid
  • Recipe Mismatch
  • Slot Information Mismatch
  • Job Type Mismatch

5、 First Glass Check

Loading Port 或 Both Port 收到 Cassette Control Command 后，Cassette Status 变为“Wait for Process”，在第一次从Cassette 带取出 Glass 前，EQP 需向BC报告“First Glass Check ” event 。

  • BC会将检查结果回复给设备。
    • 如果结果为“OK”，则可立即从 Cassette 中取出第一块 Glass ;
    • 如果结果为“NG”，设备需要自动取消 Cassette 。
  • 如果BC没有回复结果
    • 说明T1超时，设备需要重新自动重试。
