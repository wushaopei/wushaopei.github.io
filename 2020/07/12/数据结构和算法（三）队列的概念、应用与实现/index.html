5、循环队列

5.1 循环队列出现的原因：

顺序队列出队后 的空间不能再次利用，造成资源浪费。所以出现循环队列

5.2 循环队列图示

循环队列中，使用 front 来表示队首的位置，使用tail来表示队尾的位位置，当 front == tail 时，队列为空 。

常规入队：

​

在常规的循环队列中， front 队首是从第一个位置开始，依次添加元素，直到容量存满，即 tail = queue.size 。

出队：

​

出队的规则是，先进先出，即从上图的左边出队，front 队首标识向右移动，直到 front = tail 时，队列为空。

特殊情况：

​ 前面提到过，循环队列是为了解决队列出队后的空间不能再次利用而造成资源浪费。那么具体是怎么进行利用的呢？

​

如图，当队列出队两个元素后，左端出现空闲的空间，在循环队列中是可以再进行入队的。此时，tail会移动到 索引为 0 的位置，并在入队一个元素到索引为0 的地方后，将 tail 指向索引 1 的位置。

注意： 最后的空闲位置是不能存放元素的，因为 front = tail 时，队列为空的情况在该位置存放元素时会发生，导致错误异常。

所以, 此时认为队列已经满了。

所使用的公式应该是 ： tail + 1 == front 则队列满

优化过后： (tail + 1) % c == front 则队列满
