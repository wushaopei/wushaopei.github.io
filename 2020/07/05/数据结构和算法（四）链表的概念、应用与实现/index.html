一、关于数据结构的区别

数据结构有动态数据结构和静态数据结构两种.

静态数据结构：

  1. 静态数据结构有以下几种：
    1. 动态数组
    2. 栈
    3. 队列
  2. 静态数据结构的底层：其底层依托静态数组；靠 resize 解决固定容量

动态数据结构：

  1. 真正的动态数据结构有： 链表、二叉树等

    链表 – 这是本节的重点

二、什么是链表

1、链表的概念

​ 链表是最简单的动态数据结构， 其底层涉及到了引用（指针）以及大量递归的操作。

  • 链表也可以辅助组成其他数据结构

2、链表的组成结构

链表 – Linked List

说明： 链表的数据存储在“节点”（Node）中

其数据结构如下：

​ 上图是对链表结构的简单描绘，节点1 有一个next指向节点2，节点2有一个节点指向节点3，以此类推；但链表的长度是有限的，也就是说，最后一个节点必定指向值为null的next节点，如：节点3指向NULL一样。

​ 其优势： 实现了真正的动态，不需要处理固定容量的问题。

​ 但劣势（缺点）也很明显：丧失了随机访问的能力。

3、数组和链表的对比

​ 链表不能像数组一样通过索引在数组中找到对应元素，这是因为数组中的内存空间在内存里是连续分布的，可以直接寻找索引的偏移直接计算出相应的数据所存储的内存地址。所以数组这种访问的时间复杂度仅仅是O(1) 。

​ 而由于链表的每一个元素是通过 next 去一层层连接着的，这样就导致在计算机底层里链表每一个节点所在的内存的位置是不同的。我们必须通过 next 一点点的找到我们想要找的元素。所以链表这种查询的时间复杂度与链表的长度相关，即当 size 为 n 时，其时间复杂度必然为 O(n) 。

4、数组和链表的适用场景

4.1 数组
  • 数组最好用于索引有语意的情况。 即 scores[2] ，简单说就是在通过索引查询的时候。
  • 此时数组体现的优点： 支持快速查询
4.2 链表
  • 链表不适合用于索引有语意的情况
  • 最大的优点： 动态

四、链表的时间复杂度分析

1、添加操作

方法 时间复杂度
addLast(e) O(n)
addFirst(e) O(1)
add(index,e) O(n/2)=O(n)

2、删除操作

操作 时间复杂度
removeLast(e) O(n)
removeFirst(e) O(1)
remove(index,e) O(n/2)=O(n)

3、修改操作

操作 时间复杂度
set(index,e) O(n)

4、查找操作

操作 时间复杂度
get(index) O(n)
contains(e) O(n)
find(e) O(n)

说明：

在排除掉常数的情况下，链表的所有操作都是属于 O(n) 级别的时间复杂度。

但是

如果只对链表头进行操作的情况下，其时间复杂度为 O(1) ，主要包括 增、删、查三种操作。
