自动化开发（二）BC代码讲解

发表于 2020-03-24 | 分类于 自动化开发 | 阅读次数: | 阅读数

代码结构包目录图：

代码分块：

  1. api - bundle
  2. bc-common
  3. bc-model-bundle
  4. service 层： bc-service- * - * （包括bc-jobdata-service-bundle）

​ bc-service 开头的是整个BC的逻辑层代码

  1. bc-simulator-bundle （修改simulator 用的）

  2. jar 层

    包含四个模块的功能：

    bc-workflow-bundle

secsgem

plcconnector

rvconnector

  3. 日志层

    osgi-logger-common

    osgi-loglistener

  4. 数据转换层

    oee-datatransmitter-bundle

    mes-datatransmitter-bundle

    eq-datatransmitter-bundle

    edc-datatransmitter-bundle

​ secs-datatransmitter-bundle

​ fdc-datatransmitter-bundle

这一层的功能模块只做数据转换处理，转换完成后；数据会先经过bc-workflow-bundle，之后再转到bc-service- * -bundle 这一层，最终会在service层中根据 bc-model-bundle 中给定的模型将数据进行包装处理成bean形式的数据后存储起来（主要在bc-service-line-bundle中处理）。

  1. view 层 （客户端）

    client-common-bundle

    client-service-bundle

在客户端主要是通过http协议对请求进行解析，在项目中内嵌了 jetty 容器，所以项目不需要打包放到tomcat中，客户端的页面资源则需要放置到 tomcat 的webapp目录下才能访问。

api - bundle

基本原则：

  1. Port的获取

一条线上可能有多个设备，每个设备上有unit，每个unit或设备上有job。如果要找PORT，就直接找PortDAO。

Port通过属性和EQP关联起来，****

  1. 捞取数据

有一部分数据是从DB捞的，有一部分是从实时运行中捞的数据；从DB上捞的数据也就是我们做的BC工具里面的数据，我们都是放在 Data 中的。

注意：

凡是以BC * 开头的参数都是可以从Data中获取数据的，例如： BcPort、BcEquiment、BcUnit等。

这些数据都是在DB里面捞的，这些数据都是在我们 set Off 之前，key 的数据库里面的数据。

应用：

如果要查找Port的一些属性，比如： Port属于哪个设备、哪条线，都可以通过 getData 去取里面的属性就可以了。

数据实体对应关系

DB对应实体：

BcPort、BcEquiment 等Bc开头的实体都对应数据库的表，主要放在 com.glorysoft.bc.db.entity包

逻辑层对应实体：
放置在com.glorysoft.bc.entity包下

接口的两种类型：

  1. 外部系统调用BC逻辑的接口；
  2. BC逻辑处理完了给外部发送消息的接口

bc-model-bundle

前面我们知道了，API对实体的定义方式以及接口的开放类型。但那些主要还是为了给bc-model类做准备。

举例

EQP

比如我们在API中只定义了一个EQP的对象，那么在 model 启动之后，如果这条线配了四个设备，那么在model中就会有四个EQP的实体；这些实体存在于 model 中的Dao中；如下图：

在EquipmentDaoImpl 类中，存在一个 Map<String,Equipment>的集合类。如果一条线中存在有四个设备，那么这个 equipmentmap 中就有四个Equipment 。

更多的接口开放：

除了map 这一个用于存放 EQP的集合容器外，该接口类中还开放了多个给外部调用的接口，比如load（） 、getEquipmentByNo()等；

JOB

在捞取数据时，不一定就要去DB中取，也可能仅仅是从Redis中去获取。

业务场景

当所有的卡夹都全部被取出后，checkout结束后就会抛一个状态出来，这个时候我们就会把内存里面的数据包括Redis里面的数据全部删掉，该删除操作会调用JobDaoImpl中的deleteJob（） 接口。

内存问题-指南

如果程序存在内存的问题，要查看是否有glass的数据没有及时清除、残留而导致的。

Dao的封装

如图所示，所有对外提供的Dao接口类都在LineModelServiceImpl类中进行了整合，可以直接通过LineModelServiceImpl类获取所要调用的对外接口。

增量数据
