Linux（十五）服务器环境搭建-FastDFS简介与安装

发表于 2020-06-10 | 分类于 Linux ， 环境配置篇 | | 阅读次数: | 阅读数

一、传统文件上传方式缺陷

1.Web应用重新部署导致上传的文件丢失

2.集群环境下没有文件同步机制

3.扩容弹性差

4.存在单点故障

二、FastDFS功能

1.独立于应用服务器

文件本身不会在应用服务器上存储，不受应用服务器上Web应用重新部署影响。

2.内部存储服务器有数据备份

同一个文件存储在多台不同服务器上。在FastDFS内部，同一个组内的服务器存储相同文件。同组服务器之间使用同步线程同步文件数据。
避免单点故障，对外能够提供统一的访问接口。

3.动态扩容

tracker服务器管理storage服务器。动态增加storage，外界感觉不到。

三、FastDFS内在工作机制


tracker：追踪者。记录storager服务器的信息。客户端使用存储服务的时候，可以询问tracker，由tracker动态分配一个健康状态好、剩余存储空间较大的storager来提供服务。

storager：存储者。负责存储具体的文件数据。卷内有多个storager服务器，同一个卷内的storager之间会同步数据，从而保证同一个卷内各个storager服务器上存储的文件一致，避免单点故障。

tracker和storager之间通信机制：storager主动向tracker汇报。

四、FastDFS安装

源码编译安装，需要gcc-c++

1.安装FastDFS前的准备工作

①第一步：安装libevent

yum install -y libevent

②第二步：安装libfastcommon

  • 上传libfastcommonV1.0.7.tar.gz
  • 将libfastcommonV1.0.7.tar.gz移动到/usr/local目录下
  • 解压libfastcommonV1.0.7.tar.gz，进入libfastcommon-1.0.7目录
  • ./make.sh编译
  • ./make.sh install安装
  • 把/usr/lib64/libfastcommon.so文件复制到/usr/lib目录下
    • 执行复制后可用find libfastcommon.so查看是否复制到位

2.安装FastDFS主体程序

第一步：解压FastDFS tar包

tar -zxvf FastDFS_v5.05.tar.gz

第二步：进入解压目录

第三步：编译

./make.sh

第四步：安装

./make.sh install

第五步：复制配置文件

将解压目录下conf目录下的所有文件复制到/etc/fdfs目录下

3.配置tracker

创建FastDFS的家目录

mkdir -p /home/FastDFS

在tracker.conf中进行配置

vim /etc/fdfs/tracker.conf

4.配置storager

第一步：创建文件存储目录

mkdir /home/FastDFS/fdfs_storage -p

第二步：在storage.conf文件中配置

vim /etc/fdfs/storage.conf

5.上传测试

启动tracker

/usr/bin/fdfs_trackerd /etc/fdfs/tracker.conf restart

启动storager

/usr/bin/fdfs_storaged /etc/fdfs/storage.conf restart

执行测试

配置client.conf文件
vim /etc/fdfs/client.conf
