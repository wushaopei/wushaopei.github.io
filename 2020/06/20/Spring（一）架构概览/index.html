一、Spring 简史

创始人：

Rod Johnson

创始之初：

他为了证明自己的想法是正确的，在2001年10月写了一本书《Expert One-on-One J2EE》，介绍了当时Java企业应用程序开发的情况，并指出了 Java EE 和 EJB 组件框架中存在的一些主要缺陷。在这本书中，他提出了一个基于普通 Java 类和依赖注入的更简单的解决方案。在书中，他展示了如何在不使用 EJB 的情况下构建高质量、可扩展的在线座位预留系统。为了构建应用程序，他编写了超过 30,000 行的基础结构代码，项目中的根包命名为 com.interface21，所以人们最初称这套开源框架为 interface21，这就是 Spring 的前身。

以上是创始人的资深大咖（要想技术好，头顶就要秃）靓图一张。

2.2 Core Container（Spring框架的基础）

(1) Bean 模块

Beans 模块是所有应用都要用到的，包含访问配置文件，创建和管理bean，以及进行IOC和DI 操作相关的类；其中的BeanFactory 接口是核心接口，也是Spring 中的重点.

(2) Core 模块

Core 模块，是其他组件模块的基本模块”

(3) Context 模块

Context 模块，即Spring的上下文，同时也是Spring 的IOC容器。通过上下文我们可以获得容器中的Bean。该模块中的ApplicationContext 接口是Context 模块的关键，作为Beans 模块里的BeanFactory 的扩展，和 BeanFactory 的延时加载策略不同，ApplicationContext 容器实例化后会自动对所有的单实例Bean进行实例化并进行装配。

注意： Context 模块构建于 Core 和 Beans 之上。

(4) SpEL模块

SpEL模块，即Spring Expression Language ，即Spring表达式语言,是比JSP的EL更强大的一种表达式语言。SpEL支持标准数学运算符,关系运算符,逻辑运算符,条件运算符,集合和正则表达式等。它可用于将bean或bean属性注入另一个bean,还支持bean的方法调用。

2.3 AOP 层级模块

再往上一层，是AOP 层级模块，包含有AOP、Aspects、Instrumentation、Messaging四个模块。

引言：最重要的两个Spring 模块，即AOP、IOC

(1) AOP模块

在Spring 中,AOP 是以JDK \ Cglib 动态代理为基础设计出一系列方法级别的AOP 横切实现。比如：前置通知，返回通知，异常通知等。

(2) Aspects 模块

Aspects 模块集成自 AspectsJ 框架，该模块主要是为了给Spring的AOP模块提供多种AOP 的实现方法。从而在使用Spring AOP 便捷性的同时，进一步提升 Spring AOP 的功能的扩展性能。

(3) Instrumentation 模块

Instrumentation 是Spring的资源模块，区别于前面两个模块的不同，前面的两个模块是应用于方法级别的切面，而本模块主要支持对象级别的切面编程。作用：是在JVM启动时生成一个代理，通过代理来修改类的字节，从而改变一个类的功能，实现AOP的功能。其多用于面向有状态的切面编程。

(4) Messaging 模块

Messaging 模块主要是集成 MessagingAPI和消息协议提供支持，可以解读Spring 的报文。类似 TCP 的报文。

2.4 Data Access/Integration(数据访问/生成)

构成： JDBC、ORM、OXM、JMS、Transaction

(1) JDBC模块

JDBC 模块提供对jdbc 的连接的封装功能，该模块的主要意义在于简化jdbc 的使用

优化： 原生 JDBC 由一组 Java 语言组成的类和接口组成，从而为多种 关系型数据库提供访问。

而 Spring- JDBC模块则提供了对 JDBC 操作 完整封装，包括实现了一系列的模板。

(2) Transaction模块

Transaction模块：是Spring 实现 JDBC 事务控制的具体模块。该模块也是 Spring AOP的最佳实践，其对事务做了很好的封装。通过它的AOP配置可以实现一项业务的完整流程，如果业务操作失败，则整个事务回滚。

(3) ORM 模块

ORM 模块：主要集成了市面上常用的 orm 框架，包括MyBatis 、HBT等

(4) JMS模块

JMS模块：主要提供消息队列的支持

2.5 Web 组件

Web组件： 主要为 Web 项目的开发提供支持，主要包括：Spring Web、Spring WebMVC、Spring WebSocket、 Spring WebFlux四个模块

Web组件的四个模块主要为 Spring 提供了最基础的Web 支持，主要建立于Spring 的容器之上。通过 Servlet、Listen来初始化 IOC 容器。y

(1) Web 模块

Web 模块：以及简化了处理多部分请求，以及将请求参数注入到域对象的工作.

(2) WebMVC模块

WebMVC模块：该模块实现了 Spring的 model-view-controller的 Web应用。

(3) WebSocket模块

WebSocket模块：WebSocket是html下的一种全双工通信协议，建立联系后，webSocket的服务端和客户端都可以主动向对方发送、接受数据。

与http的比较：http具有无连接、不可靠、尽最大努力的特点；而websocket可以认为是 http 为了支持长连接而打的一个大补丁。

WebSocket 内部的消息体依赖于 Messaging 模块

(4) WebFlux 模块

WebFlux 是 Spring 5 推出的响应式流的响应式模块，可以用来建立异步、非阻塞、事件驱动的服务，扩展性也比较好。
